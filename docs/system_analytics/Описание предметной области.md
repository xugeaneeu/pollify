Система голосования/сбора мнений предназначена для организации и проведения опросов среди пользователей с целью получения обратной связи, принятия решений или анализа мнений.

#### Текущий процесс:
На данный момент опросы могут проводиться:
	- вручную (устно, на собраниях)
	- через мессенджеры или соцсети
	- с помощью разрозненных инструментов (Google Forms, таблицы и т.д.)

#### Основная бизнес-функция: 
Cбор голосов

#### Бизнес-процессы:
1. Создание опроса (анонимный |  не анонимный)
	1. Инициатор проходит авторизацию в системе
	2. Инициатор переходит в раздел создания опроса
	3. Инициатор вводит основные параметры:
		1. название опроса
		2. описание ???
		3. формулировку вопроса
		4. варианты ответов
	4. Инициатор указывает настройки опроса:
		1. тип опроса: анонимный или не анонимный
		2. срок проведения (дата начала и окончания)
		3. доступ (открытый или для ограниченного круга пользователей)
	5. Система выполняет валидацию введённых данных
	6. Система регистрирует опрос в базе данных
	7. Система присваивает ему уникальный идентификатор
2. Голосование
	1. Пользователь проходит авторизацию в системе
	2. Переходит к списку доступных опросов 
	3. Выбирает конкретный опрос (доступ по ссылке)
	4. Система проверяет: доступен ли опрос (по времени и правам доступа); голосовал ли пользователь ранее
	5. Отказ или опрос
	6. Система фиксирует голос:
		1. Для не анонимного опроса: сохраняет связь «пользователь — выбранный вариант»
		2. Для анонимного опроса: сохраняет только выбранный вариант, отдельно фиксирует факт участия пользователя (без сохранения выбора)
3. Просмотр результатов
	1. Пользователь или администратор выбирает опрос
	2. Система проверяет: завершён ли опрос или разрешён ли просмотр результатов до завершения, есть ли у пользователя права на просмотр
	3. Система формирует результаты на основе сохранённых данных
	4. Клиент отображает: общее количество голосов, распределение голосов по вариантам, процентное соотношение
